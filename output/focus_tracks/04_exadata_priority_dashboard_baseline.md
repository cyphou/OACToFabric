# Exadata Priority Dashboard Baseline (Draft)

Date: 2026-06-01
Status: Draft - pending business approval

## Purpose

Provide a Sprint 1 candidate list for Track B3 so Exadata-backed OAC migration can move from generic discovery scope to dashboard-level execution.

## Selection Rules

1. Executive or finance dashboards first.
2. Dashboards with direct Exadata subject-area dependency first.
3. Dashboards with high user reach or monthly close dependency first.
4. Dashboards with limited custom visual complexity preferred for Wave 1.

## Draft Priority Buckets

### Priority 1 - Must migrate first

1. Executive KPI dashboard
2. Revenue and margin dashboard
3. Monthly close / finance variance dashboard
4. Sales pipeline summary dashboard

### Priority 2 - Important but not gating

1. Regional performance dashboard
2. Product profitability dashboard
3. Customer segment dashboard

### Priority 3 - Defer after parity sign-off

1. Operational detail dashboards
2. Analyst self-service boards
3. Low-usage historical dashboards

## Required Metadata per Dashboard

1. OAC path
2. Business owner
3. Exadata schema / subject area
4. Refresh criticality
5. Must-match visuals and filters
6. Sign-off owner

## Approval Checklist

- [ ] Priority 1 list approved by business owner
- [ ] Exadata source domains mapped per dashboard
- [ ] Sign-off owner assigned per dashboard
- [ ] Deferred list accepted
